13-mongo-typeorm

mongo typeorm

Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3000

In the inventory below, Wired (2) means Nest actively loads that class; Example (0) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

13-mongo-typeorm — NestJS Sample

MongoDB persistence via TypeORM (not Mongoose). A minimal Photo read API showing @ObjectIdColumn entities and repository injection against Mongo.

Quick start

cd sample/13-mongo-typeorm
npm install
docker-compose up -d    # MongoDB on :27017
npm run start:dev

App listens on http://localhost:3000.

Method Path Description
GET /photo List photos
