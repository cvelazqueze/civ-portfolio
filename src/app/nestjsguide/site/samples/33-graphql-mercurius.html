33-graphql-mercurius

graphql mercurius

Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule, new FastifyAdapter())
Inventory kind
HTTP server

Default port: 3000

Also configured: HTTP adapter: **Fastify** · GraphQL endpoint enabled

Global setup in main.ts: `ValidationPipe` (global, `@nestjs/common`)

In the inventory below, Wired (1) means Nest actively loads that class; Example (1) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

33-graphql-mercurius — NestJS Sample

Code-first GraphQL on Fastify using the Mercurius driver (not Apollo/Express). Queries, mutations, subscriptions, GraphiQL, and a custom Date scalar.

Quick start

cd sample/33-graphql-mercurius
npm install
npm run start:dev

GraphQL / GraphiQL: http://localhost:3000/graphql

Generated schema: schema.gql
