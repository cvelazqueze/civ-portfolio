dynamic modules

Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3000

In the inventory below, Wired (3) means Nest actively loads that class; Example (0) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

25-dynamic-modules — NestJS Sample

Build a custom dynamic module with ConfigModule.register(options) — configurable at import time via a custom provider token and @Inject.

Quick start

cd sample/25-dynamic-modules
npm install
npm run start:dev

App listens on http://localhost:3000.

Dynamic module pattern

// config.module.ts
static register(options: ConfigModuleOptions): DynamicModule {
  return {
    module: ConfigModule,
    providers: [
      { provide: CONFIG_OPTIONS, useValue: options },
      ConfigService,
    ],
    exports: [ConfigService],
  };
}
