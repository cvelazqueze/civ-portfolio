12-graphql-schema-first

graphql schema first

Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3000

Also configured: GraphQL endpoint enabled

Global setup in main.ts: `ValidationPipe` (global, `@nestjs/common`)

In the inventory below, Wired (3) means Nest actively loads that class; Example (2) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

12-graphql-schema-first — NestJS Sample

GraphQL schema-first API with Apollo driver. SDL lives in .graphql files; TypeScript types are generated into graphql.schema.ts. Includes queries, mutations, subscriptions, field resolvers, guards, and a custom directive transformer.

Quick start

cd sample/12-graphql-schema-first
npm install
npm run start:dev

GraphQL Playground: http://localhost:3000/graphql

Regenerate TS typings from SDL (when schema changes):

npm run generate:typings
