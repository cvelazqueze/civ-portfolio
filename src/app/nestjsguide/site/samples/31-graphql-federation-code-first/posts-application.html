31-graphql-federation-code-first / posts-application

posts-application

Understanding this sample

This page documents a sub-application inside sample 31-graphql-federation-code-first (part of a multi-service setup such as GraphQL federation). Each sub-app has its own main.ts and module tree.

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3003

Also configured: GraphQL endpoint enabled

In the inventory below, Wired (1) means Nest actively loads that class; Example (2) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

Posts Application — Federation Sample 31

Posts subgraph — owns Post and extends User with a posts field.

Quick start

cd sample/31-graphql-federation-code-first/posts-application
npm install
npm run start:dev

GraphQL: http://localhost:3003/graphql
