Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3000

In the inventory below, Wired (2) means Nest actively loads that class; Example (0) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

08-webpack — NestJS Sample

Minimal hello-world app demonstrating Webpack Hot Module Replacement (HMR) for Nest development. Code changes reload without restarting the full Node process when run via the webpack dev script.

Quick start

cd sample/08-webpack
npm install
npm run start:dev    # webpack + HMR watch

Standard build (no HMR):

npm run build
npm run start:prod

App listens on http://localhost:3000.

Method Path Response
GET / Hello world!

How the app boots

HMR block in main.ts:

if (module.hot) {
  module.hot.accept();
  module.hot.dispose(() => app.close());
}

Mental model

  1. Normal nest build compiles with tsc — no HMR.
  2. Webpack dev bundles the app and swaps modules on change via module.hot.
  3. app.close() on dispose prevents orphaned servers when HMR reloads.
