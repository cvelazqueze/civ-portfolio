Understanding this sample

Standalone HTTP application

This sample starts a normal web server. External clients reach it via URLs (browser, curl, mobile app).

When this shape is used: Best when you expose a REST/JSON API, MVC pages, Swagger, or file uploads from one deployable Node app.

Bootstrap
NestFactory.create(AppModule)
Inventory kind
HTTP server

Default port: 3000

Global setup in main.ts: `ValidationPipe` (global, `@nestjs/common`)

In the inventory below, Wired (1) means Nest actively loads that class; Example (0) means the file is reference-only until you register it in a module or decorator.

Related concepts: application type explained · Wired vs Example · Why decorators? · Networking acronyms (TCP, RPC…)

21-serializer — NestJS Sample

Response serialization with class-transformer and Nest's ClassSerializerInterceptor. Demonstrates @Exclude, @Expose, and @Transform on entity classes.

Quick start

cd sample/21-serializer
npm install
npm run start:dev

App listens on http://localhost:3000.

Method Path Response (serialized)
GET / User without password; fullName exposed; role as string

Example JSON shape:

{
  "id": 1,
  "firstName": "Kamil",
  "lastName": "Mysliwiec",
  "fullName": "Kamil Mysliwiec",
  "role": "admin"
}

How the app boots

Entity transformation rules

Field / property Decorator Library Effect
password @Exclude() class-transformer Omitted from response
fullName @Expose() getter class-transformer Computed field in output
role @Transform(...) class-transformer { id, name } → "admin"

AppController returns new UserEntity({ ... }) — serialization runs on class instances.
