NestJS Samples — Learning Guide

A beginner-friendly walkthrough of every official Nest sample. Each example is its own page with diagrams, inventories, and plain-language explanations of wired vs example code, decorators, and whether the app is a standalone HTTP server, microservice, or something else.

42 pages · fully offline · copy the whole site/ folder anywhere

New to NestJS?

Start with the Glossary (Nest concepts + acronyms like TCP, RPC, JWT, GraphQL), then open 01-cats-app — it is the reference HTTP app with modules, controllers, services, guards, and interceptors explained end to end.

How to read each sample page

  1. Understanding this sample — what kind of app it is and why it bootstraps that way.
  2. Core elements inventory — tables of modules, controllers, guards, etc. Focus on Wired rows first.
  3. Diagrams — request flow and module relationships (already rendered; no live Mermaid needed offline).
  4. Decorator glossary — which @ symbols appear and whether they come from Nest, your code, or a library.

Start here

The best samples to read first. They show the core NestJS building blocks before transports, databases, or GraphQL enter the picture.

01-cats-app cats app The canonical “first Nest app”: modules, controllers, services, guards, pipes, and interceptors — with clear wired vs example files. 18-context context No HTTP server at all — `createApplicationContext` for scripts, CLI, or background workers. 25-dynamic-modules dynamic modules Modules configured at runtime with `.forRoot()` / `.forFeature()`.

HTTP & web servers

Classic request/response APIs and server-side rendering. These apps listen for HTTP and return JSON, HTML, or files.

10-fastify fastify Same Nest patterns but Fastify as the HTTP adapter (faster than Express). 15-mvc mvc Server-rendered HTML views (Handlebars) — MVC style. 17-mvc-fastify mvc fastify MVC rendering on top of Fastify. 24-serve-static serve static Serving static front-end files from the same Nest process. 11-swagger swagger OpenAPI/Swagger UI generated from decorators.

Real-time & streaming

WebSockets, Server-Sent Events, and long-lived connections instead of one-shot HTTP requests.

02-gateways gateways HTTP + Socket.IO in one process; shows how a WebSocket gateway sits beside REST. 16-gateways-ws gateways ws Native WebSocket gateway (`@WebSocketGateway`) without Socket.IO. 28-sse sse Server-Sent Events for one-way streaming to the browser.

Microservices & messaging

Apps that talk over TCP, gRPC, NATS, Redis, MQTT, or queues — not only HTTP. Includes hybrid apps that run HTTP and a message transport together.

03-microservices microservices Hybrid app: HTTP on port 3001 plus a TCP microservice in the same Node process. 04-grpc grpc Pure gRPC microservice using `@GrpcMethod` instead of HTTP routes. 26-queues queues Background jobs with Bull queues and `@Processor`. 30-event-emitter event emitter In-process pub/sub with `@nestjs/event-emitter`.

Databases & persistence

Connecting Nest to SQL or MongoDB through TypeORM, Sequelize, Mongoose, or Prisma.

05-sql-typeorm sql typeorm PostgreSQL via TypeORM entities and `@nestjs/typeorm`. 06-mongoose mongoose MongoDB with Mongoose schemas inside Nest modules. 07-sequelize sequelize SQL database through Sequelize models. 13-mongo-typeorm mongo typeorm TypeORM connected to MongoDB (document store via ORM). 14-mongoose-base mongoose base Minimal Mongoose setup without extra Nest wrappers. 22-graphql-prisma graphql prisma GraphQL resolvers backed by Prisma ORM.

GraphQL

Schema-first and code-first GraphQL APIs, federation across multiple services, and Fastify/Mercurius integration.

12-graphql-schema-first graphql schema first GraphQL with a `.graphql` schema file and generated types. 23-graphql-code-first graphql code first GraphQL schema built from TypeScript decorators (`@ObjectType`, `@Query`). 31-graphql-federation-code-first graphql federation code first Multiple GraphQL subgraphs merged by a federation gateway (code-first). 32-graphql-federation-schema-first graphql federation schema first Federation with SDL schema files across gateway and subgraphs. 33-graphql-mercurius graphql mercurius GraphQL on Fastify using the Mercurius driver.

Security, caching & serialization

Authentication, response shaping, and performance helpers that wrap around your handlers.

19-auth-jwt auth jwt JWT authentication with Passport strategies and guards. 20-cache cache Response caching with `@nestjs/cache-manager`. 21-serializer serializer Class-transformer interceptors to shape JSON output.

Build tooling & module systems

Webpack, Babel, ESM packages, and hot module replacement — how the sample is bundled and run, not a Nest feature per se.

08-webpack webpack Custom Webpack build for the Nest app. 09-babel-example babel example Transpiling with Babel instead of plain `tsc`. 34-using-esm-packages using esm packages Consuming native ESM npm packages from a Nest app. 35-use-esm-package-after-node22 use esm package after node22 ESM interop patterns on modern Node. 36-hmr-esm hmr esm Hot module replacement during development with ESM.

Other patterns

File uploads, cron jobs, and smaller focused demos.

27-scheduling scheduling Cron and interval tasks with `@nestjs/schedule`. 29-file-upload file upload Multipart file uploads with Multer integration.
